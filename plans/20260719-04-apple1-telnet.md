# Apple 1 Over Telnet: `ewm one --tty` + systemd Socket Activation

- **Design docs:** `notes/APPLE1.md` (the machine: PIA keyboard in /
  display out — the property that makes this mode natural) and
  `notes/REMOTE.md` (the sibling track: this is the character-stream
  answer to what RFB does with pixels). This plan carries its own
  design details below — it is experiment-sized; as-built notes go to
  `notes/APPLE1.md` if it graduates.
- **Status:** draft — iterate here before kickoff
- **Target:** `main`; PR granularity decided at kickoff

## Goal

```
telnet retro.example.com 6502
```

…and you are at a cold-booted Woz Monitor. systemd owns the socket
(inetd-style `Accept=yes`): every connection starts a fresh headless
machine whose keyboard and display are the connection; hangup kills
it. The same mode with no socket at all is an Apple 1 in your local
terminal:

```
ewm one --config builtin:apple1 --tty        # stdin/stdout is the terminal
nc -l 6502 --sh-exec 'ewm one --tty'         # poor man's socket test
```

The emulator does **no networking**: systemd passes the accepted
connection as stdin/stdout (`StandardInput=socket`). One mode, three
uses — local terminal, `nc`, systemd telnet farm.

## Why this fits

- The Apple 1 *is* a terminal machine — one PIA, bytes in, bytes out
  (`notes/APPLE1.md`). A TCP stream is a more faithful display than a
  framebuffer; the VNC route (REMOTE Phase 7) renders a *picture* of a
  terminal.
- `ewm/examples/one.rs` already proves the headless shape (machine +
  `drain_display()` + uppercase + paced keys); this productizes it.
- Everything the machine needs is embedded (builtin configs, builtin
  ROMs), so the served process can run in a near-empty sandbox.

## Design

### The loop (T1)

- **No SDL.** `--tty` short-circuits `one::main` after `parse_options`
  / `build_machine`: no window, no renderer, no `Tty` texture.
- **Wall-clock pacing**: step `ONE_CPS / 50` cycles per 20 ms tick
  (the machine must run at 1.023 MHz, not host speed — Integer BASIC
  at native speed is the whole charm). Drain the display every tick.
- **Input** on a reader thread feeding an `mpsc` channel (house style:
  blocking I/O + threads). The main loop injects **one byte at a time,
  only when the PIA's previous byte has been consumed** (the latch is
  one byte deep; the ROM polls `$D011` — same problem `two`'s
  `RemoteKeys` solved for VNC). a–z uppercase on the way in, as the
  SDL frontend and the C original do; the Control key maps naturally
  (Ctrl-A … Ctrl-Z arrive as `0x01`-`0x1A` and pass straight through —
  the Apple 1 keyboard *had* CTRL, so control bytes belong to the
  machine).
- **Output**: display bytes masked to 7 bits (the model does this),
  CR → CRLF for terminals/telnet, printables passed through.
- **Session end**: stdin EOF or stdout write error (client hung up) →
  clean process exit; systemd reaps the instance.

### Reset: Meta-R (and telnet BREAK)

The real machine had RESET and CLR SCREEN buttons next to the
keyboard — out-of-band controls, not keys. Over a byte stream the
emulator needs an out-of-band-ish escape, and the constraint is that
**Ctrl anything is machine input** (real CTRL key), and **bare ESC is
machine input too** — it is the Woz Monitor's cancel-line key. What
the Apple 1 keyboard did *not* have is Meta/Alt:

- **Meta-R = reset.** Terminals with "Alt/Option sends Escape" (xterm
  default, iTerm/Terminal.app option, most Linux terminals) transmit
  Meta-R as `ESC` `r` back-to-back. Disambiguation from a bare ESC
  meant for the monitor: hold a received ESC for a short window
  (~50 ms); if `r`/`R` arrives within it (in practice: the same TCP
  segment), reset the machine; anything else (or the timer expiring)
  forwards the held ESC followed by the pending byte. This is exactly
  how terminal applications have always told ESC from escape
  sequences.
- **Telnet BREAK = reset** too (T2): `IAC BRK` / `IAC IP` — the
  authentic serial-terminal "attention" — is what `telnet`'s
  `send brk` emits. Two spellings, one action.
- Reset = `cpu.reset()` (warm, like Cmd-R); the monitor prints its
  prompt again. No screen clear — the terminal scrolls, which is
  period-correct for a printing/dumb terminal.

### Telnet niceties (T2)

Raw telnet defaults to line mode with local echo; the monitor wants
character-at-a-time and echoes itself. At session start send
`IAC WILL ECHO` + `IAC WILL SGA`; afterwards parse inbound `IAC`
sequences: strip/refuse negotiations (`DONT`/`WONT` anything else),
map `BRK`/`IP` to reset, drop the rest. ~40 lines on `std`, in house
style; `nc` and local-terminal users never see any of it (negotiation
only ever *responds* to telnet clients — no, correction: we *send*
the two WILLs unconditionally in telnet mode). **Decision at
kickoff**: make telnet negotiation part of `--tty` always, or a
`--tty telnet` variant with plain `--tty` staying byte-clean for
local terminals. Recommendation: always send the two WILLs only when
the first inbound bytes contain `IAC` (auto-detect) — local terminals
stay clean, telnet clients negotiate; simplest rule that serves both.

### systemd (T2)

```
# ewm-one.socket
[Socket]
ListenStream=6502
Accept=yes
MaxConnectionsPerSource=4

# ewm-one@.service
[Service]
ExecStart=/usr/local/bin/ewm one --config builtin:apple1 --tty
StandardInput=socket
DynamicUser=yes
ProtectSystem=strict
ProtectHome=yes
PrivateTmp=yes
NoNewPrivileges=yes
RuntimeMaxSec=4h
```

Committed under `scripts/systemd/` with a README stanza. The binary
needs no filesystem (embedded configs + ROMs), so the sandbox can be
tight. `RuntimeMaxSec` stops abandoned sessions from accumulating
1 MHz loops; `MaxConnectionsPerSource` keeps it a toy, not a botnet
amplifier.

## Phases

| Phase | Description | Size | Status |
|---|---|---|---|
| T1 | `--tty` mode: headless loop, wall-clock pacing, latch-paced input, CR→CRLF, uppercase, ESC/Meta-R reset, EOF exit — testable over in-memory streams | M | Done |
| T2 | Telnet IAC handling + BREAK reset; `scripts/systemd/` units; docs (README Experimental entry) | S/M | Not started |

- **T1 gate:** the loop takes `Read`/`Write` generics, so tests drive
  it over pipes with no terminal: boot to the `\` prompt; type
  `E000.E00F` and assert the BASIC dump (mirroring `one_boot.rs`);
  Ctrl bytes pass through; bare ESC cancels a monitor line; `ESC r`
  resets (prompt again); EOF ends the loop. Manual observation:
  `ewm one --tty` in a real terminal reaches BASIC via `E000R`.
  Standard gates.
- **T2 gate:** IAC unit tests (negotiation reply, stripping, BRK →
  reset, auto-detect leaves nc streams untouched); a scripted
  `nc`-pair end-to-end check; manual observation: real `telnet
  localhost 6502` session against a locally started socket unit (or
  `systemd-socket-activate` for the gate on macOS-less CI — decide
  what is practical at kickoff). Standard gates + README check.

## Hazards

- **The ESC hold window** is the one subtle piece: it must never eat
  a bare ESC (monitor line-cancel is a real workflow). The window
  applies only when nothing follows in the buffer; tests pin both
  paths.
- **Pacing under load**: the 20 ms tick must not busy-wait; sleep on
  the tick, `mpsc::recv_timeout` for input. A disconnected client
  must not leave the loop spinning (write error → exit).
- **`--tty` + `--print-config`** and friends must keep composing (the
  mode flag is pass-2, like `--serve`).
- The SDL frontends are untouched — the golden-BMP suite is, as
  always, the tripwire.

## Decisions to make at kickoff

1. **Flag spelling** — `--tty` (recommended) or `--serve tty` for
   symmetry with `two`'s `--serve`.
2. **Telnet negotiation** — auto-detect on first `IAC`
   (recommended), always-on, or a separate `--tty telnet` variant.
3. **A quit chord** — Meta-Q to hang up from the emulator side, or
   rely on the client (telnet `Ctrl-]` `quit`; recommended: rely on
   the client, fewer stolen keys).
4. **The example** — retire `ewm/examples/one.rs` in favor of the
   real mode (recommended), or keep it as the minimal illustration.
5. **PR granularity** — per phase or one PR.

## Backlog (recorded, out of scope)

- **`two` over tty** — the ][+ is a framebuffer machine; a `--tty`
  for it would need VT100 screen-diff tricks. Not this experiment.
- **Session banner** — a short pre-boot banner line (machine name,
  Meta-R hint) before the monitor prompt; trivial but decide if the
  purism police allow it.
- **A public instance** — rate limiting, abuse posture, logging;
  operational, not emulator work.
